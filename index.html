SWAG Improvement Proposals

SWAG Improvement Proposals (SWIPs) describe standards for the SWAG.FINANCE platform, including core protocol specifications, client APIs, and contract standards.

Contributing

  1. Review SWIP-0.
  2. Clone the repository
  3. Add your SWIP to your version of the repository. There is a template SWIP here..
  4. Submit a Pull Request to SWAGâ€™s SWIPs repository.

Your first PR should be a first draft of the final SWIP. An editor will manually review the first PR for a new SWIP and assign it a number before merging it. Make sure you include a discussions-to header with the URL to a discussion forum or open GitHub issue where people can discuss the SWIP as a whole.

When you believe your SWIP is mature and ready to progress past the DRIP phase, you should ask to have your issue added to the next governance meeting where it can be discussed for inclusion in a future platform upgrade. If the community agrees to include it, the SWIP editors will update the state of your SWIP to Approved.

SWIP status terms

  • DRIP - a SWIP that is still being drafted.
  • Proposed - a SWIP that is ready to be reviewed in a governance call.
  • Approved - a SWIP that has been accepted for implementation by the SWAG community.
  • Implemented - a SWIP that has been released to mainnet.
  • Rejected - a SWIP that has been rejected.
